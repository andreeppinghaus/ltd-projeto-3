										                                                                                                                                                           Biblioteca de Phyton e C	                  
								                                                                                                                                                                        Objetivo
                                                                      O objetivo deste projeto é desenvolver uma biblioteca para Arduino que permita a comunicação com sensores, utilizando tanto a linguagem de                                                                               programação C quanto Python no computador. A biblioteca facilitará a integração de diferentes componentes de hardware em projetos Arduino e                                                                              oferecerá uma interface amigável para interação com o usuário.

1.1 Planejamento

1.1.1 Recursos Humanos
⦁	Desenvolvedores: Serão necessários desenvolvedores com experiência em linguagem C para programação do firmware do Arduino e em Python para a interface do usuário.
⦁	Gerente de Projeto: Um gerente de projeto será responsável por coordenar as atividades, definir metas, acompanhar o progresso e garantir que o projeto seja concluído dentro do prazo estabelecido

1.1.2 Recursos Materiais
⦁	Hardware: Será necessário hardware para o desenvolvimento e teste da biblioteca, incluindo placas Arduino, sensores e componentes eletrônicos diversos.
⦁	Computadores: Cada membro da equipe de desenvolvimento precisará de um computador para escrever código, realizar testes e documentação.
⦁	Software: Ferramentas de desenvolvimento, como IDEs para Arduino (como Arduino IDE ou PlatformIO) e editores de código para Python, serão necessárias. Além disso, software para documentação, controle de versão e comunicação entre a equipe também será essencial.

1.1.3 Recursos Temporais
⦁	Tempo da Equipe: O tempo dos membros da equipe dedicado ao projeto deve ser estimado e planejado adequadamente, levando em consideração outras responsabilidades e projetos em andamento.
⦁	Prazos: Serão estabelecidos prazos para cada fase do projeto, incluindo desenvolvimento, testes, documentação e lançamento.
⦁	Contingências: Deve-se incluir uma margem de tempo para imprevistos e atrasos que possam surgir ao longo do projeto.

1.1.4 Comunicação e Colaboração

⦁	Ferramentas de Comunicação: Serão utilizadas ferramentas de comunicação online, como e-mails, mensagens instantâneas e videoconferências, para facilitar a colaboração entre os membros da equipe, especialmente se estiverem trabalhando remotamente.
⦁	Plataforma de Gerenciamento de Projetos: Uma plataforma de gerenciamento de projetos, GitHub, pode ser utilizada para acompanhar o progresso das tarefas, atribuir responsabilidades e manter a equipe informada sobre o status do projeto.

1.2 Escopo

1.2.1 Entregaveis
	- Trabalhar para modelar, extrair, tratar e construir uma API para atender o objetivo;
	- Pesquisar os sensores;
    	- Criar códigos isolados nas linguagens Python e em C++;
   	- Criar uma classe para cada linguagem;
   	- Disponiliblizar no github.

1.2.2 Membros
O grupo de alunos foi dividido em equipes como documentação e desenvolvimento de códigos simulando o trabalho em uma empresa do setor de software.
